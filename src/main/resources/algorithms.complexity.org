
Сложность алгоритмов (в порядке роста сложности):
N - размер задачи
K - константа

| Класс | Название         | Комментарий                                                                                           |
|-------+------------------+-------------------------------------------------------------------------------------------------------|
| 1     | Константный      | Выборка элементов из массива, хэш таблицы                                                             |
| logN  | Логарифмический  | Такая зависимость появляется в результате сокращения размера задачи на каждом шаге итерации алгоритма |
| N     | Линейный         |                                                                                                       |
| NlogN | Квазилинейный    |                                                                                                       |
| N^2   | Квадратичный     | Два вложенных цикла (не всегда)                                                                       |
| N^K   | Полиномиальный   | K вложенных циклов                                                                                    |
| 2^N   | Экспоненциальный | Рекурсия (не всегда). Обработка всех подмножеств некоторого множества, которое состоит из элементов   |
| N!    | Факториал        | Рекурсия внутри цикла. Перестановки.                                                                  |
